## أفكاري للبنود الناقصة (بناء إنتاجي)

### 1) رسم دائرة النطاق على الخريطة (map radius select)
- Leaflet: زر «حدّد نطاق» → `L.circle(center, radius)` قابل للسحب/التكبير + مقبض نصف قطر · إرسال `{lat,lng,radiusKm}` للـviewport → السيرفر يفلتر بـ`ST_DWithin`/Haversine.
- الموبايل: `SearchResultsMap` يمرّر النطاق للـ`GET /search/map` (يحتاج param `radius_km`+center) + عرض عدد النتائج داخل الدائرة.
- GPS: «نطاق حولي» = دائرة حول موقعي الحالي (locate موجود، أوسّعه).

### 2) دورة حياة استيراد السيارات (import lifecycle)
- الاستيراد ليس مجرد فلتر — له **حالات متابعة**: `requested → sourcing → customs → shipping → delivered`.
- backend: `import_requests` أو حالة على الإعلان + إشعارات لكل انتقال (تربط بطبقة الإشعارات الموجودة).
- الموبايل: قسم/تبويب متابعة استيراد داخل السيارات (منفصل عن التصفّح العادي) — نفس منطق دورة حياة الحجز.
- **قرار مالك:** هل الاستيراد B2C (المستخدم يطلب سيارة مستوردة ونتابعها) أم B2B (تجار الاستيراد)؟ يحدّد الحقول.

### 3) البنود الموجودة — عمق المراجعة المقترح
- **الأصوات:** كود سليم → المطلوب device-test بعد النشر + التأكد أن `SoundContext` مفعّل افتراضياً + إعداد كتم.
- **الإشعارات + الكاليندر + AI:** مراجعة رحلة كاملة (سيناريو-بسيناريو) بعد سحب main على Replit.
